1. 工程化ai项目

3. 生产级服务（http连接池+超时控制）
4. 便于测试化

## 练习任务

1. ai工程化（cmd + internal=ai+transport）
2. 流式输出（sse）
3. 超时控制 & 请求取消
4. 接口范型
5. http客户端（生产级）
6. json复杂解析（流式json复杂解析）
7. 测试（不做测试，做利于测试的架构设计）

## 练习思路

1. ai工程化：cmd + internal=ai(ask ai)+transport(http req/resp)+llm(types)
2. 流式输出（sse）：生产者（ai大模型/模拟ai大模型输出）、消费者（for range channel to read）、协议（sse）
   (主流程发起请求之后，另起一个 Goroutine 持续读取流式响应，再通过 Channel 把结果逐步交给调用方。)
   生产者负责 close(channel)，且用 defer 兜底。消费者用 for range 自动在关闭时退出。谁创建谁关闭、绝不在消费侧关闭。
   发送也要走 select + ctx.Done()。如果只 out <- chunk 而消费者已经不读了，这个 goroutine 会永久阻塞泄漏。Agent 里这种泄漏极隐蔽且致命。
   错误在同一个 channel 中返回，而不是另开一个 error channel。单通道 + 互斥字段，消费侧逻辑最简单。
3. context 控制：超时取消（整体调用链路超时时间）+ 手动取消（用户取消比如读取某信号 + 上游判断后取消）
4. 统一请求响应结构 & 抽象 Provider & 范型增强复用性（这个需要多明白一下，go的范型特性）
   - 统一请求响应结构：所有请求响应都有抽象概念，抽出共有的。根据特性添加其他的
   - 抽象 Provider 就是抽象各家大模型，一般抽象方法：名称、非流式请求、流式请求
   - 范型增强复用：（这个需要多明白一下，go的范型特性）
5. 生产级http客户端：
   - 客户端配置（DialTimeout + KeepAlive + MaxIdleConns + MaxIdleConnsPerHost + IdleConnTimeout + TLSHandshakeTimeout）
   - 高并发&连接池复用（
     - http.Transport 连接池复用、TCP/TLS 握手
     - http.Client 底层 HTTP 执行，不设全局超时
     - 自定义 Client	统一封装限流 + 指数退避重试 + 抖动
     - context 单次请求生命周期控制（超时 / 取消）
   ）
   - 非流式重试逻辑和令牌桶限流策略（最好可以：横切关注点(cross-cutting concern)的封装，让调用方无感）
6. 流式json复杂解析
   - 延迟解析 json.RawMessage（json.Decoder + DisallowUnknownFields 生产解析不开启）
   - 自定义 UnmarshalJSON 方法应对"同一字段可能是多种结构"(多模态：文字、视频等)的业务场景（反序列化串行尝试，序列化反向解析，减少无效解析）
   - 安全序列化：就是对 json 序列化时，如果出现错误不 panic，而是投出错误
   - 本项目实现总结：
     - 延迟解析：新增 `ToolCall.Args json.RawMessage` 和 `ParseRawInto[T]`，先保留工具参数原始 JSON，等确认工具类型后再解析成具体结构。
     - 严格解析：新增 `ParseStrict[T]`，内部使用 `json.Decoder` 和 `DisallowUnknownFields`，用于测试或强约束场景发现未知字段；生产默认 `ParseInto[T]` 保持宽松解析，兼容厂商扩展字段。
     - 联合类型：新增 `MessageContent`，让 `Message.Content` 同时支持纯文本字符串和多模态数组；`UnmarshalJSON` 先尝试字符串，再尝试 `[]ContentPart`，都失败才返回明确错误。
     - 反向序列化：`MessageContent.MarshalJSON` 有分块时输出数组，否则输出字符串，和反序列化逻辑保持对称，便于请求和响应无缝往返。
     - 多模态分块：新增 `ContentPart` 和 `MediaURL`，目前覆盖 `text`、`image_url`、`video_url`，后续音频等类型可以按同样结构扩展。
     - 安全序列化：新增 `SafeMarshal`，序列化失败时返回带上下文的 error，不在底层工具函数里 panic。
